# Contributing to Gordon

We love your input! We want to make contributing to this project as easy and transparent as possible.

## Pull Request Process

1. Ensure any install or build dependencies are removed before the end of the layer when doing a build.
2. Update the README.md with details of changes to the interface, this includes new environment variables, exposed ports, useful file locations and container parameters.
3. Increase the version numbers in any examples files and the README.md to the new version that this Pull Request would represent.
4. Write clear, well-documented code inside the `/src` folder.
5. All scripts go into `/scripts`.
6. Add unit tests in `/tests` for any Python code you add.
7. Linting runs automatically via GitHub Actions against `/src`.
8. You may merge the Pull Request in once you have the sign-off of other maintainers.
